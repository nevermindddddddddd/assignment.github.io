Statistics Table

Purpose & Requirements

The Statistics Table page presents key data on hunger and food security in a structured, accessible format. The challenge was to make complex data easy to understand while ensuring the table remained accessible and responsive across devices.

Technical Implementation

Key HTML elements and techniques used:

  • Semantic Table Structure - Used proper HTML table elements (thead, tbody, th) with appropriate scope attributes
  • Table Caption - Added a descriptive caption for screen readers
  • Row and Column Spanning - Implemented colspan and rowspan to create a more complex table structure
  • Visual Hierarchy - Used background colors and typography to create visual hierarchy within the table
  • Responsive Design - Ensured the table remained usable on smaller screens

Table Structure Implementation

<table class="stats-table" aria-label="Food security statistics by region">
  <caption class="table-caption">Global Food Security Comparison (2023-2024)</caption>
  <thead>
      <tr>
          <th scope="col"><img src="/placeholder.svg?height=30&width=30" alt="Region" title="Geographic Region">Region</th>
          <th scope="col"><img src="/placeholder.svg?height=30&width=30" alt="Undernourished" title="Percentage of Population Undernourished">Undernourished (%)</th>
          <!-- Additional columns -->
      </tr>
  </thead>
  <tbody>
      <tr>
          <td colspan="5" class="merged-cell">Developing Regions</td>

  </tbody>
</table>

Challenges & Lessons Learned

Challenges Faced

  1. Table Responsiveness: Making complex tables responsive while maintaining data relationships was challenging.
  2. Design System Consistency: Ensuring consistent application of the design system across all pages required careful planning.
  3. Performance Optimization: Balancing visual richness with performance, especially for image-heavy pages.

Solutions & Lessons

  1. For tables, I implemented horizontal scrolling on small screens rather than restructuring the data, which preserved the data relationships.
  2. Created a comprehensive CSS variable system and component library to maintain consistency.
  3. Optimized images and implemented lazy loading for better performance.

These challenges taught me the importance of planning the information architecture before implementation and the value of creating reusable components for consistency and efficiency.

Accessibility

I prioritized accessibility in my implementations through the following measures:

  • Semantic HTML: Used appropriate HTML elements to provide structural meaning.
  • ARIA Attributes: Added ARIA labels and descriptions where necessary, especially for tables and interactive elements.
  • Color Contrast: Ensured all text meets WCAG AA contrast requirements against its background.
  • Keyboard Navigation: Made all interactive elements accessible via keyboard.
  • Alternative Text: Provided descriptive alt text for all images.
  • Table Accessibility: Used proper table markup with captions, headers, and scope attributes.

All pages were tested using the WAVE accessibility evaluation tool and keyboard navigation to identify and resolve potential issues.

Compliance

The pages I implemented comply with Janet regulations governing web page publication in the following ways:

  • Valid HTML5: All pages have been validated using the W3C Markup Validation Service.
  • Accessibility: Pages meet WCAG 2.1 Level AA standards for accessibility.
  • Responsive Design: All pages are responsive and function properly across different devices and screen sizes.
  • Cross-Browser Compatibility: Pages have been tested and function correctly in major browsers.
  • Performance: Pages load efficiently with optimized assets.
  • Content Standards: All content is appropriate, accurate, and properly attributed.
