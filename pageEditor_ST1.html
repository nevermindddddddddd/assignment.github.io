Overview

As Student 1 on this project, I was responsible for implementing the following pages:

  • Splash Screen (Validation)
  • Volunteer Page (Validation)
  • Content Page: Food Security Programs (Validation)

Each page was developed using a combination of HTML and CSS, with a focus on accessibility, user experience, and adherence to the global styling guidelines established for the website.

Splash Screen

Purpose & Requirements

The splash screen serves as the entry point to our website, providing visitors with a brief introduction to our initiative and team. The page automatically redirects to the home page after 4 seconds.

Technical Implementation

Key HTML elements and techniques used:

  • Meta Refresh Tag: Used the HTML meta refresh tag to automatically redirect the user to the home page after 4 seconds.
  • Background Image: Applied a full-screen background image with an overlay to enhance readability of text content.
  • CSS Animation: Created a custom loading animation using CSS keyframes to indicate the page is loading.

HTML Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="refresh" content="4;url=home.html"> <!-- Automatic redirect after 4 seconds -->
    <title>Zero Hunger Initiative - Splash</title>
    <link rel="stylesheet" href="style.css">
    <style>
        /* CSS styles for splash screen */
    </style>
</head>
<body>
    <div class="splash-content">
        <h1 class="splash-title">Zero Hunger Initiative</h1>
        <p class="splash-mission">Mission statement...</p>
        
        <div class="team-members">
            <h2>Our Team</h2>
            <p>Student 1, Student 2, Student 3, Student 4</p>
        </div>
        
        <div class="loader"></div> <!-- Animated loading indicator -->
        <p>Redirecting to home page...</p>
    </div>
</body>
</html>

CSS Animation

I created a loading animation using CSS keyframes to indicate that the page is transitioning:

.loader {
    margin-top: 2rem;
    position: relative;
    width: 80px;
    height: 80px;
    margin: 0 auto;
}

.loader::after {
    content: "";
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    border: 5px solid var(--neutral-300);
    border-radius: 50%;
    border-top-color: var(--primary);
    animation: spin 1s ease-in-out infinite;
}

@keyframes spin {
    to {
        transform: rotate(360deg);
    }
}

Volunteer Page

Purpose & Requirements

The Volunteer page serves as a hub for users to explore volunteering opportunities, read reviews from past volunteers, and submit their own feedback. It was designed to be interactive and informative.

Technical Implementation

Key HTML elements and techniques used:

  • Interactive Dropdown: Implemented a styled dropdown menu for sorting or filtering content.

Challenges & Lessons Learned

Challenges Faced

  1. Responsive Image Sizing: Ensuring images displayed properly across different screen sizes required careful implementation of CSS properties.
  2. CSS Animation Compatibility: Creating animations that worked consistently across different browsers required testing and refinement.
  3. Internal Navigation Layout: Designing a navigation system that remained accessible while not taking up too much screen space was challenging.

Solutions & Lessons

  1. Used the object-fit property to maintain image aspect ratios while ensuring proper sizing within containers.
  2. Implemented simpler animations with wider browser support and tested across multiple devices.
  3. Created a compact navigation system that collapses on smaller screens to maintain usability while preserving space.

These challenges taught me the importance of testing across different devices and browsers, as well as the value of progressive enhancement to ensure core functionality remains accessible to all users.

Accessibility

I prioritized accessibility in my implementations through the following measures:

  • Semantic HTML: Used appropriate HTML elements to provide structural meaning (headings, sections, etc.).
  • Alt Text: Added descriptive alt text for all images to support screen readers.
  • Color Contrast: Ensured sufficient contrast between text and background colors to enhance readability.
  • ARIA Labels: Added ARIA attributes where necessary to improve screen reader navigation.
  • Keyboard Navigation: Ensured all interactive elements are accessible via keyboard.
  • Responsive Design: Implemented responsive layouts that work across different screen sizes and devices.

All pages were tested using WAVE (Web Accessibility Evaluation Tool) to identify and resolve potential accessibility issues.

Compliance

The pages I implemented comply with Janet regulations governing web page publication in the following ways:

  • Valid HTML5: All pages have been validated using the W3C Markup Validation Service to ensure compliance with HTML5 standards.
  • Accessibility: Pages meet WCAG 2.1 Level AA standards for accessibility, ensuring they are usable by people with various disabilities.
  • Responsive Design: All pages are responsive and function properly across different devices and screen sizes.
  • Security: No client-side scripts are used that could pose security risks.
  • Copyright: All images used are either placeholders or properly attributed royalty-free resources.
  • Privacy: The pages do not collect any personal information without explicit consent.

These measures ensure that the pages meet the technical and ethical standards required for academic web publishing.
