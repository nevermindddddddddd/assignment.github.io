Design Process

For each page I implemented, I followed a structured design process:

  1. Research: Understanding user needs and goals related to the specific page functionality
  2. Information Architecture: Organizing content in a logical, intuitive structure
  3. Wireframing: Creating low-fidelity layouts to establish content hierarchy and user flow
  4. Visual Design: Applying the established design system while ensuring optimal usability
  5. Implementation: Coding the design with a focus on accessibility and responsiveness
  6. Testing: Validating the implementation for usability and accessibility

Feedback Form Implementation

Purpose & Requirements

The feedback form was designed to collect user input about the Zero Hunger Initiative's programs and website. It needed to be comprehensive yet user-friendly, with appropriate form validation and clear instructions.

UX Considerations

Key UX principles applied to the feedback form:

  • Clear Instructions: Provided context and guidance at the beginning of the form
  • Logical Grouping: Organized form fields into related sections using fieldsets and legends
  • Visual Hierarchy: Used typography and spacing to establish clear relationships between elements
  • Input Assistance: Added labels, placeholders, and help text to guide users
  • Error Prevention: Implemented client-side validation with clear error messages
  • Responsive Design: Ensured the form works well on all device sizes

Technical Implementation

Key HTML elements and techniques used:

  • Semantic Form Elements: Used appropriate form controls with proper attributes
  • Fieldsets: Grouped related form elements for better organization
  • Input Types: Utilized various HTML5 input types (email, date, time, etc.) for better user experience
  • Form Validation: Implemented HTML5 validation attributes and custom JavaScript validation
  • Character Counter: Added a dynamic character counter for the feedback message

Accessibility Improvements

As the UX Specialist, I conducted a comprehensive accessibility audit of the website and implemented various improvements to ensure the site is usable by people with diverse abilities and needs.

Keyboard Navigation

âœ“

Alternative Text

Added descriptive alt text for all images, and appropriate ARIA labels for interactive elements.

âœ“

Form Accessibility

Implemented proper form labeling, error handling, and instructions to ensure forms are accessible to screen reader users.

âœ“

Responsive Design

Specific Improvements

Key accessibility improvements implemented across the website:

  • Skip Navigation Link: Added a visually hidden skip link that becomes visible on focus, allowing keyboard users to bypass navigation
  • ARIA Attributes: Added appropriate ARIA roles, states, and properties to enhance screen reader compatibility
  • Focus Management: Implemented proper focus management for interactive components
  • Form Error Handling: Created accessible error messages for form validation
  • Keyboard Support for Interactive Elements: Added JavaScript to ensure custom interactive elements work with keyboard navigation

Challenges & Lessons Learned

Challenges Faced

  1. Form Validation Accessibility: Creating form validation that was both user-friendly and accessible to screen reader users required careful consideration.
  2. Interactive Elements: Ensuring custom interactive elements (like the team member cards) were fully accessible via keyboard and screen readers was challenging.
  3. Content Organization: Structuring complex nutritional information in a way that was both comprehensive and digestible required multiple iterations.

Solutions & Lessons

  1. Implemented both visual and programmatic error messages for form validation, with clear instructions on how to correct errors.
  2. Used a combination of ARIA attributes, keyboard event listeners, and focus management to make interactive elements accessible.
  3. Applied content chunking principles and visual hierarchy to break down complex information into manageable sections.

These challenges reinforced the importance of considering accessibility from the beginning of the design process, rather than as an afterthought. I also learned the value of testing with actual assistive technologies rather than relying solely on automated testing tools.
